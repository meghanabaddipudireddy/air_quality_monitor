# air_quality_monitor
This project is a battery-powered air quality monitor built on an STM32L476 microcontroller running FreeRTOS. It reads CO₂, temperature, and humidity from a Sensirion SCD40 and particulate matter (PM2.5) from a Plantower PMS5003, calculates a real-time AQI score, logs every reading to SPI NOR flash storage, and transmits data wirelessly to a phone via a Nordic nRF52840 BLE module — all managed by five concurrent FreeRTOS tasks.

# Hardware 
STM32L476RG — The brain of the system. Every other component is a passive peripheral that waits to be commanded. The L4 series was chosen for its ultra-low-power modes (down to 3µA in STOP2) while still having enough performance to run FreeRTOS, a hardware FPU for AQI calculations, and six simultaneous hardware peripherals (I2C, SPI, two UARTs, DMA, ADC).

Sensirion SCD40 — A true photoacoustic CO₂ sensor. Unlike cheap alternatives like the MQ-135 which estimate CO₂ from VOC measurements, the SCD40 uses an IR LED and microphone to directly measure CO₂ concentration with ±50 ppm accuracy. It also provides temperature and humidity with built-in compensation. Communicates over I2C.

Plantower PMS5003 — A laser scattering particle counter that measures PM1.0, PM2.5, and PM10 concentrations. Unlike I2C sensors which require a master to request data, the PMS5003 streams 32-byte frames every second automatically over UART — captured by STM32 DMA without waking the CPU.

Winbond W25Q32 — 4MB SPI NOR flash used for data logging. Chosen over an SD card because NOR flash has no filesystem overhead, predictable 1-5ms write latency, and fits within the 128KB SRAM budget. A wear-leveling scheme distributes writes across all 512 sectors to maximize flash lifetime.

Nordic nRF52840 — A dedicated BLE coprocessor. The STM32L476 has no built-in radio, and offloading BLE to a separate chip keeps the real-time RTOS scheduling clean. The STM32 sends 12-byte UART packets to the nRF52840 which handles all BLE stack complexity internally.
e-ink display — Chosen over LCD or OLED because it consumes power only during a refresh (~15mA for ~2 seconds) and holds its image at zero power. At a 25-minute update interval this makes the display essentially invisible in the power budget.

LiPo + TP4056 + AMS1117 — The LiPo cell stores energy at 3.7V. The TP4056 safely charges it from USB-C using CC/CV algorithm. The AMS1117 LDO regulates the cell's swinging 3.0-4.2V output to a stable 3.3V rail for all components.

# System Architecture

The firmware is organized into four layers. Each layer only communicates with the layer directly above or below it.

main.c            — initializes drivers, creates tasks, starts scheduler
tasks             — sensor_task, pipeline_task, flash_task, ble_task
drivers           — scd40, pms5003, w25q32, eink
STM32 HAL         — HAL_I2C, HAL_SPI, HAL_UART (generated by CubeMX)
hardware          — physical chips on the board

Drivers wrap all hardware communication. A task never touches I2C registers or SPI chip-select pins directly — it just calls scd40_read() or w25q32_write_record() and gets results back.

Tasks implement the application logic using FreeRTOS queues to pass data between them:
- sensor_task (priority 4) — reads SCD40 over I2C and PMS5003 over UART every 5 seconds, sends combined readings to xSensorQueue
- pipeline_task (priority 3) — receives from xSensorQueue, calculates AQI, packs a log_record_t, fans out to xFlashQueue and xBLEQueue
- flash_task (priority 2) — receives from xFlashQueue, writes 32-byte records to W25Q32 NOR flash
- ble_task (priority 1) — receives from xBLEQueue, sends a 12-byte UART packet to the nRF52840 BLE module

# Protocols
I2C — Used by the SCD40. A two-wire master-slave protocol (SDA + SCL) where the STM32 initiates every transaction by addressing the device (0x62) before sending or receiving data. Multiple devices can share the same two wires. The SCD40 uses clock stretching — it holds the clock line low while preparing measurement data, stalling the STM32 until it's ready.

SPI — Used by the W25Q32 flash and e-ink display. A four-wire protocol (MOSI, MISO, SCK, CS) where the STM32 selects a device by pulling its chip-select pin low. SPI is significantly faster than I2C — the W25Q32 runs at up to 104 MHz — making it suitable for bulk data transfers like writing log records or sending a 5KB display framebuffer. Both devices share the same SPI bus with separate chip-select pins.

UART — Used by the PMS5003 and nRF52840. An asynchronous two-wire protocol where both sides agree on baud rate in advance with no clock wire. The PMS5003 streams 32-byte frames at 9600 baud continuously without being asked — captured by STM32 DMA into a ring buffer without waking the CPU. The nRF52840 receives 12-byte packets at 115200 baud from the STM32 and broadcasts them over BLE to a connected phone.
